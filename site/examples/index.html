    CMSAPI Examples

    Python-based REST API for retrieving, analyzing and consolidating content published in UniBE CMS and other information systems.

    OpenAPI Docs

    • v3 generated by Swagger / ReDoc
    • v2 generated by Swagger (deprecated)

    Data retrival

    The endpoints which are backed by a PostgreSQL database (object relation mapping) support the query parameters offset and limit for pagination.

    To filter the response values, e.g. date_after or sections can be used.

    Furthermore lang is used for language specific queries.

    Example

    Content retrival

    The ZMS Content endpoint provides instant retrival of content stored in ZODB by path or uuid query parameters.

    It utilizes the ZMS default XML export function xml = zodb.toXml(deep=True|False). The boolean parameter deep controls the processing of children in the content tree.

    The default XML response is automatically converted into JSON using xmltodict Python library xmltodict.parse(xml).

    Caveat

    The current implementation is a simple solution to proof the concept.

    These response schemes need to be revised and adapted based on concrete use cases — especially for better client-side processing of multilingual content.

    Cluster deployment

    The CMSAPI stack consists mainly of two services:

    • a few API containers which provide the endpoints
    • a single ADM container which processes the object relation mapping based on cron jobs during office hours by /app/bin/python -m cmsapi.admin.main update --metaobj all >> /app/log/api-cron.log

    The database connection strings are set in the environment variables ZODB_STORAGE and SQLDB_STORAGE.

    Example

      api-v3:
    image: ep-devops.id.unibe.ch:5000/id/unibe-cmsapi-v3:rc-3.0.0
    volumes:
      - logs:/app/log
    environment:
      - PYTHONUNBUFFERED=true
      - ZODB_STORAGE=zeo://idcmsdata3.unibe.ch:7001  # at www.cmstest.unibe.ch (synced daily with PROD data)
      - SQLDB_STORAGE=postgresql://pgtest.unibe.ch:5432/unibe_cmsapi?sslmode=require&application_name=CMSAPI
      - FORWARDED_ALLOW_IPS=*  # trust proxy headers of Kemp => https https://www.uvicorn.org/deployment/
    networks:
      - worker
      - cms_network
    secrets:
      - source: test-unibe-cmsapi-v3-sqldb-credentials
        target: sqldb.credentials
    deploy:
      replicas: 3
      restart_policy:
        condition: on-failure
      resources:
        limits:
          cpus: "1.0"
          memory: 768M
      labels:
        - "traefik.enable=true"
        - "traefik.docker.lbswarm=true"
        - "traefik.docker.network=worker"
        - "traefik.http.services.test-unibe-cmsapi.loadbalancer.server.port=5003"
        - "traefik.http.services.test-unibe-cmsapi.loadbalancer.passhostheader=true"
        - "traefik.http.routers.test-unibe-cmsapi.rule=(Host(\"api.cms.test.unibe.ch\") && PathPrefix(\"/v3\")"
        - "traefik.http.routers.test-unibe-cmsapi.entrypoints=websecure"
        - "traefik.http.routers.test-unibe-cmsapi.tls=true"
    ports:
      - 5003

      adm-v3:
    image: ep-devops.id.unibe.ch:5000/id/unibe-cmsapi-v3:rc-3.0.0
    volumes:
      - logs:/app/log
    environment:
      - PYTHONUNBUFFERED=true
      - ZODB_STORAGE=zeo://idcmsdata3.unibe.ch:7001  # at www.cmstest.unibe.ch (synced daily with PROD data)
      - SQLDB_STORAGE=postgresql://pgtest.unibe.ch:5432/unibe_cmsapi?sslmode=require&application_name=CMSAPI
      - RUN_CRONJOBS=true  # init_scripts startUp in this dedicated admin container only
    networks:
      - worker
      - cms_network
    secrets:
      - source: test-unibe-cmsapi-v3-sqldb-credentials
        target: sqldb.credentials
    deploy:
      replicas: 1
      restart_policy:
        condition: on-failure
      resources:
        limits:
          cpus: "1.0"
          memory: 768M
